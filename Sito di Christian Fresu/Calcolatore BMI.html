Introduzione

A cosa serve un calcolatore di BMI?

Un “calcolatore BMI” serve per calcolare l’Indice di Massa Corporea, ovvero un dato biometrico che si ottiene dal rapporto tra il peso e il quadrato dell’altezza:

BMI=peso/(statura2)BMI = peso / (statura^2)BMI=peso/(statura2)﻿

(Dove “peso” è in Kg e “statura” in m)

Il suo valore determina lo “stato” di una persona:

CASI POSSIBILI

CATEGORIABMI RANGE
Sottopeso severo< 16,5
Sottopesoda 16,5 a 18,4
Normaleda 18,5 a 24,9
Sovrappeso da 25 a 30
Obesità di I gradoda 30,1 a 34,9
Obesità di II gradoda 35 a 40
Obesità di III grado> 40

Perché programmare un calcolatore di BMI?

Questo è stato il mio primo progetto in Flutter e siccome lo reputo un esercizio molto “scolastico” (in quanto l’ho replicato anche in C e in Java all’Accademia Digitale Liguria) ho pensato che fosse un buon punto di partenza vista anche la logica e le funzioni non troppo complesse; in queste condizioni ho potuto concentrarmi sulla realizzazione di un Layout che combinasse più widget (di cui avevo solo una conoscenza teorica e poco pratica).

Come si è evoluto il progetto

Inizialmente il progetto prevedeva solo dei ‘TextField’ (widget che si occupa di prendere i valori di input inseriti dall’utente) e un bottone che calcolasse il BMI, restituendo un messaggio di riepilogo con un alert. Le implementazioni che sono avvenute a progetto iniziato sono state:

  • L’inserimento di un bottone nell’AppBar* per ripulire tutti i campi di input
  • L’inserimento di un bottone (sempre nell’AppBar) per modificare la modalità di input, ovvero per passare da ‘TextField’ a ‘Slider’**

All’interno del progetto

Quali funzioni sono state definite?

_calcoloBMI

  • Return Type: double
  • Parametri: double statura, double peso
  • Descrizione: Il compito di questa funzione è semplicemente quello di calcolare il BMI con i dati inseriti dall’utente (i parametri). Il risultato viene troncato a due cifre dopo la virgola grazie al metodo qua linkato:

  • Codice
    //Funzione che esegue il calcolo
  double _calcoloBMI(double statura, double peso) {

    double result = 0;
    statura /= 100; //Conversione da cm a m
    result = peso / (statura*statura);

    return double.parse((result).toStringAsFixed(2)); //Risultato restituito con solo 2 decimali dopo la virgola
  }

_esitoFinale

  • Return Type: String
  • Parametri: double _valBMI
  • Descrizione: Questa funzione prende il valore del BMI (calcolato dalla funzione sopra citata e salvato in una apposita variabile) e restituisce la “zona” a cui appartiene l’utente. Quindi se per esempio il BMI arrivato come parametro avrà un valore pari a 17, questa funzione restituirà ‘Sottopeso’
  • Codice
    //Funzione che restituisce la zona di appartenenza (a seconda del valore del BMI)
  String _esitoFinale(_valBMI) {

    if(_valBMI < 16.5) {
      return 'Sottopeso severo';
    }

    if(_valBMI >=16.5 && _valBMI <= 18.4) {
      return 'Sottopeso';
    }

    if(_valBMI >= 18.5 && _valBMI <= 24.9) {
      return 'Normale';
    }

    if(_valBMI >= 25 && _valBMI <= 30) {
      return 'Sovrappeso';
    }

    if(_valBMI >= 30.1 && _valBMI <= 34.9) {
      return 'in Obesità di I grado';
    }

    if(_valBMI >= 35 && _valBMI <= 40) {
      return 'in Obesità di II grado';
    }
      return 'in Obesità di III grado';

  }

_clearInput

  • Return Type: void
  • Parametri: --
  • Descrizione: Questa funzione ha il compito di ripulire tutti i campi di input. Viene richiamata quando viene cliccato il bottone che si trova nell’AppBar

  • Codice
    //Esito che ripulisce l'input
  void _clearInput() {

    if(!_sliderVisibile) {
      //Vengono ripuliti i 'TextField'
      controllerPeso.clear();
      controllerAltezza.clear();

      //Il valore di 'peso' e 'altezza' vengono resettati
      setState(() => _valPeso = 0);
      setState(() => _valAltezza = 0);
    }

  }

_switchSlider

  • Return Type: void
  • Parametri: --
  • Descrizione: I ‘TextField’ vengono sostituiti da degli ‘Slider’. Ciò avviene (e quindi la funzione viene invocata) quando viene cliccato l’apposito bottone situato nell’AppBar, vicino a quello che serve per pulire gli input

  • Codice
    void _switchSlider() {

    if(_sliderVisibile) {
      setState(() => _sliderVisibile = false);
      setState(() => _valPeso = 0);
      setState(() => _valAltezza = 0);

    }

    else {

      setState(() => _sliderVisibile = true);
      setState(() => _valPeso = _valSliderPeso);
      setState(() => _valAltezza = _valSliderAltezza);

    }

  }

Widget utilizzati

In quanto primo progetto tutti i widget sono risultati nuovi, ma in particolare vorrei sottolineare l’utilizzo dei seguenti:

Tooltip

Corrisponde a una piccola etichetta che compare su computer al passaggio del cursore su un’icona e su telefono con una pressione prolungata sempre sull’icona. L’obiettivo e l’utilità di questo widget è quella di spiegare brevemente con poche parole a cosa serve quel determinato bottone

Visibility

Tramite l’utilizzo di una variabile booleana permette di vedere/non vedere un determinato widget all’interno dell’interfaccia. In questo caso, la sua utilità si è potuta vedere nello switch da TextField a Slider e viceversa.

ButtonTheme

All’interno di un SizedBox e genitore di un ElevatedButton, questo widget permette di personalizzare un bottone (scegliere larghezza, altezza, ecc.)

AlertDialog

Restituisce come output un alert, ovvero un riquadro contenente del testo. Questo widget è diviso in tre sezioni:

  • Titolo
  • Contenuto
  • Bottone

Palette dei colori utilizzati

  • Colore primario (sfondo) ⇒ F3FFF7
  • Colore secondario (App Bar) ⇒ 42E8C0
  • Colore Testo ⇒ 00000 (nero)

Dependencies utilizzate

Le dependencies sono pacchetti necessari per svolgere alcune azioni come ad esempio cambiare il colore di sfondo o l’icona all’avvio dell’applicazione.

In questo progetto ho inserito la dependency necessaria per trattare (come ho detto nell’esempio) le proprietà all’avvio dell’app:

  • flutter_native_splash: ^2.2.8

Note & Appunti

In questa sezione vorrei trattare alcune delle tante nozioni che ho appreso nella realizzazione di questa semplice applicazione

  • Ogni TextField necessita di un suo controller e di una sua variabile che verifichi se l’input è corretto (se così non fosse, entrambi gli input dovrebbero essere corretti per evitare un messaggio di errore). Oltre a ciò, con l’utilizzo di un singolo controller, le azione eseguite su una casella di input si verificherebbero anche su tutte le altre (inserimento o rimozione del testo, messaggi di errore, ecc.)

Ambienti di sviluppo utilizzati

  • Android Studio ⇒ La totalità del lavoro (scrittura codice e build dell’APK) è stata svolta su questo ambiente di sviluppo
