Procedura

  • Aprire Android Studio
  • Creare un nuovo progetto di Flutter
  • Cliccare su File → New → New Flutter Project
  • Settare nome applicazione, scegliere il percorso del Flutter SDK e settare il nome del dominio della company

  • main.dart

    import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  //Classe fondamentale per il funzionamento dell'app

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Hello World Demo Application',
      theme: ThemeData (
        primarySwatch: Colors.blue,
      ),
      home: MyHomePage(title: 'Home page'),
    );
  }
}

class MyHomePage extends StatelessWidget {
  MyHomePage({Key? key, required this.title}) : super(key: key);
  final String title;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text(this.title),
      ),
      body: Center (
        child:
          Text (
            'Hello World',
          )
      ),
    );
  }
}

Struttura dell’applicazione

  • Dopo aver avviato l’applicazione il file presenterà la seguente struttura

Di seguito, l’elenco delle cartelle e della loro funzione all’interno del progetto

  1. Android ⇒ Contiene un codice sorgente auto generato, necessario per creare l’applicazione Android
  1. ios ⇒ Contiene un codice sorgente auto generato, necessario per creare le applicazioni iOS
  1. lib ⇒ La cartella principale in cui è contenuto il codice in Dart. Questo codice è scritto utilizzando il Flutter Framework
  1. lib/main.dart ⇒ Marca l’Entry Point dell’applicazione in Flutter
  1. test ⇒ Questa cartella contiene il codice Dart necessario per testare l’applicazione Flutter
  1. test/widget_test.dart ⇒ Codice Dart sample
  1. .gitignore ⇒ Questo è un file di testo che contiene una lista di file, estensioni di file e cartelle che dovrebbero essere ignorate quando si lavora con Git
  1. .metadata ⇒ Questo file è gestito automaticamente da Flutter ed è usato per tracciare tutte le proprietà del progetto
  1. .packages ⇒ Questo file viene generato automaticamente dal Flutter SDK e contiene una lista di dependencies per il progetto
  1. .iml ⇒ Questo file viene nominato in accordo con il nome del progetto. Contiene diverse impostazioni del progetto
  1. Pubspec.yaml ⇒ File di configurazione del progetto. Questo contiene:
    • Proprietà generali come: impostazioni, nome, descrizione, ecc.
    • Dependencies
    • Assets (ad esempio immagini)

Struttura codice

In questa sezione si analizzerà il codice riportato sopra (”main.dart”)

import 'package:fluttermaterial.dart';

  • Questa linea importa il pacchetto ‘material’
  • Questo pacchetto crea interfacce in accordo con le guide linea messe a disposizione da Android

void main() => runApp(MyApp());

  • Questa linea rappresenta l’Entry Point dell’applicazione
  • Richiama la funzione runApp e gli passa un oggetto della classe MyApp
  • runApp ha il compito di inserire sullo schermo il Widget che viene passato come parametro

class MyApp extends StatelessWidget {
  //Classe fondamentale per il funzionamento dell'app

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Hello World Demo Application',
      theme: ThemeData (
        primarySwatch: Colors.blue,
      ),
      home: MyHomePage(title: 'Home page'),
    );
  }
}

  • I Widgets sono usati per creare l’interfaccia nel Flutter Framework
  • StatelessWidget è un widget, che come si può intuire dal nome, non mantiene nessuno stato del widget
  • Facendo l’estensione della classe (e di seguito l’override), viene sovrascritto il metodo di costruzione del widget
  • L’obiettivo di del metodo ‘build’ è quello di creare una parte della UI nell’applicazione
  • Il metodo ‘build’ usa MaterialApp
  • Serve per creare il primo livello della UI
  • Possiede 3 proprietà: title (il titolo dell’applicazione), theme (il tema cromatico del widget), home (ci dice che abbiamo settato un altro widget. In questo caso MyHomePage)

class MyHomePage extends StatelessWidget {
  MyHomePage({Key? key, required this.title}) : super(key: key);
  final String title;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text(this.title),
      ),
      body: Center (
        child:
          Text (
            'Hello World',
          )
      ),
    );
  }
}

  • MyHomePage è uguale a MyApp
  • Questa classe, però a differenza di quella precedente, restituisce uno Scaffold Widget
  • Lo Scaffold è usato per creare UI con un design conforme e ha due proprietà importanti:

  1. appBar ⇒ Mostra l’header dell’applicazione
  1. body ⇒ Mostra il contenuto dell’applicazione
