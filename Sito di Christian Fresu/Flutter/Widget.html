Introduzione

  • Come già detto nella pagina precedente, un’applicazione è essa stessa un widget, composta da tanti sub-widgets

ALBERO DEI WIDGET

Un Widget può:

  • Definire il design di qualcosa
  • Stampare a schermo qualcosa
  • Può gestire interazioni, ecc.

Sintassi per definire un widget

  • Per definire un widgte, bisogna usare la seguente sintassi
Class ImageWidget extends StatelessWidget {

	//Istruzioni della classe

}

Tipologie di widget

Esistono 2 tipi di widgets nel Flutter Framework:

  • Widget visibili
  • Widget invisibili

I primi gestiscono i dati di input & output, mentre i secondi gestiscono il layout e i controlli dell’applicazione

Widget VISIBILI

  • Gestiscono i dati di input & output
  • I 4 widget (visibili) più comuni sono:

  1. Image
  1. Icon
  1. Text
  1. Button

TEXT

  • Questo widget mostra del testo
  • Text può essere allineato tramite la proprietà ‘textAlign’
  • Text può essere personalizzato usando la proprietà ‘Style’

new Text(
	'Hello World';
	textAlign: textAlign.Center,
	Style: new textStyle(fontWeight: Fontweight.bold),
	)

BUTTON

  • In Flutter esistono due tipi di ‘Button’ (Flat Button & Raised Button)
  • Raised Button ⇒ Dà l’impressione che il tasto sia sopra alla superficie
  • Flat Button ⇒ Dipende dal colore che viene dato alla superficie, per dare all’utente l’impressione di bottone

new FlatButton(

	child: Text("Click Here"),
	onPressed: () {

		//Istruzioni

	},
)

IMAGE

  • Questo widget permette di visualizzare immagini
  • Le immagini possono essere reperite da diverse sorgenti
  • Per inserire un’immagine nel progetto, prima di tutto bisogna inserire un Assets Folder (qui verranno tenute le immagini)
  • Per creare un Assets Folder bisogna andare nel file “pubspec.yaml” e inserire il seguente codice

assets:
	-assets/

  • Dopodiché aggiungere ‘Image.asset’ (’assets/computer.png’) nel codice Dart

ICON

  • Agisce come un container per, appunto, icone

new Icon(
	Icons.add,
	size: 34.0,
)

Widget INVISIBILI

  • Gestiscono il Layout & i controlli
  • I 4 widget (invisibili) più comuni sono

  1. Column
  1. Row
  1. Centre
  1. Padding

COLUMN

  • Inserisce i widget in verticale
  • Gestisce lo spazio tra i widget utilizzando le proprietà ‘mainAxisAlignment’ (asse verticale) e ‘crossAxisAlignment’ (asse orizzontale)

new Column(

	mainAxisAlignment: MainAxisAlignment.center,

	children<Widget> [

		new Text(
			"Elemento 1",
		),

		new Text(
			"Elemento 2",
		),
	],
),

ROW

  • Inserisce i widget in orizzontale
  • Gestisce lo spazio tra i widget utilizzando le proprietà ‘mainAxisAlignment’ (asse verticale) e ‘crossAxisAlignment’ (asse orizzontale)

new Row(

	mainAxisAlignment: MainAxisAlignment.spaceEvenly,
	children<Widget> [

		new Text(
			"Elemento 1",
		),

		new Text(
			"Elemento 2",
		),
	],
),

PADDING

  • Crea una sorta di spazio (settato manualmente) tra i widget
  • Si può settare un Padding in tutte le direzioni (Top, Left, Rigth, Bottom)
