Access Contol Modifier

  • Parole chiave usate per limitare lo scopo di una classe, di dati, di un metodo o di un costruttore
  • Lo scopo definisce da dove queste entità possono avere accesso e dove no
  • Esistono 4 Access Control Modifier in Java:

  1. Default
  1. Public
  1. Private
  1. Protected

1. Default

  • Quando non è presente nessun Access Modifier. Per usare ‘default’, non bisogna esplicitarlo

Esempio

class Esempio {

	//Istruzioni

}

2. Public

  • Quella parte di codice è accessibile ovunque. Per definirla si sua la parola chiave ‘public’

Esempio

public class Esempio {

	public int n = 10;

}

  • Se ‘n’ non fosse stato ‘public’, altre classi avrebbero avuto accesso alla classe ‘Esempio’ ma non a ‘n’

3. Private

  • Accessibile solo nella stessa classe. Per dichiarare del codice ‘private’ , si usa la parola chiave ‘private’

Esempio

class esempio {

	private int n = 10;

}

  • Si può accedere ad ‘n’ solo nella classe ‘Esempio’

4. Protected

  • Si può accedere a quei dati solo quando una classe richiama la classe ‘Esempio’
  • In questo caso solo i metodi, le variabili e i costruttori possono essere dichiarati ‘protected’. Le classi no
  • Per definire questo Access Control, si usa la parola chiave ‘protected’

Esempio

class Esempio {

	protected int n = 10;

}
