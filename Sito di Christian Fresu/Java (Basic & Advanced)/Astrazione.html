Astrazione

Implementare l’astrazione
Esempio
  • Processo in cui si nascondono i dettagli di implementazione all’utente (mostrando solo le funzionalità)
  • In poche parole si nascondono i dettagli di background
  • La sicurezza di un programma potrebbe essere a rischio se non c’è astrazione

Implementare l’astrazione

  • L’astrazione si può effettuare mediante interfacce e classi astratte

  • CLASSE ASTRATTA ⇒ Classe incompleta che va estesa se si vuole usare; la sua estensione la renderà completa
  • Una classe astratta si dichiara usando la parola chiave ‘abstract’
  • Una classe astratta può contenere o non contenere metodi astratti
  • La catena di classi astratti ad un certo punto deve essere rotto. Si deve presentare una classe concreta che definisca tutti i metodi astratti

  • METODO ASTRATTO ⇒ Metodo senza un corpo. Viene dichiarato in una classe astratta e il suo corpo viene creato quando si estende la classe astratta di appartenenza
  • Una classe con almeno un metodo astratto, deve essere dichiarata come ‘abstract’

Esempio (dichiarazione classe astratta)

public abstract class Name {

	//Istruzioni

}

Esempio (estensione della classe astratta)

public class ExampleClass extends Name {

	//Istruzioni

}

Esempio (metodo astratto)

public abstract returnType methodName();

  • La definizione di un metodo astratto si trova all’interno della classe figlio (lì viene definito il corpo)

Esempio

  • Classe astratta ⇒ “EsseriViventi”
  • Metodi astratti ⇒ “respirazione ()”, “riproduzione()”

Nel momento in cui istanzio un oggetto della classe figlia “AnimaleTerrestre”, andrò a settare i metodi astratti “respirazione ()” e “riproduzione ()”. Allo stesso modo per gli altri animali (classi)

Questi metodi (e anche questa classe), partono astratti in quanto non è possibile definire dei valori di default per essi
