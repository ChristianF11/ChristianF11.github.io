Gestione delle eccezioni

  • L’eccezione interrompe la normale esecuzione del codice
  • Le eccezioni si attivano quando si verifica un errore dell’utente, del programmatore o relativo alle risorse fisiche
  • exception handling ⇒ Mantiene il normale flusso del programma

  • Le eccezioni si dividono in due categorie:

  1. Checked
  1. Unchecked

Checked exceptions

  • Chiamate anche ‘compile - time exceptions’
  • Si attivano quando si verifica un errore in esecuzione
  • Il programma ha bisogno di gestire queste eccezioni

Unchecked exceptions

  • Chiamate anche ‘run-time exceptions’
  • Sono errori logici o bug di programmazione
  • Gli errori NON sono eccezioni (gli errori non possono essere gestiti)

Try, catch, finally

  • try ⇒ All’interno di questo blocco viene inserito il comando (o i comandi) da testare

  • catch / finally ⇒ Vengono inseriti (il finally è facoltativo, mentre il catch no) dopo il blocco try
  • Possono esserci più catch dopo try
  • finally ⇒ Compare sempre dopo try e catch (a prescindere dall’esito)

Esempio

try {

	//Blocco da testare

}

catch (Exception e) {

	System.out.println(e.getMessage()); 
	//Se è stato settato un messaggio; altrimenti si può inserire qui

}

finally {

	//Messaggio che verrà visualizzato a prescindere dall'esito try-catch

}

Throw

  • Si usa per “lanciare” un’eccezione in maniera esplicita nel metodo in cui si pensa si possa verificare

Esempio

public void metodoEsempio () throws Exception {

	if(/*condizione*/) 
		throw new Exception (/*messaggio*/);

	//Resto del body

}
